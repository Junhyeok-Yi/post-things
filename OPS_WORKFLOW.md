# Post-Things 운영 규약 v1

## 목적
주간 개발/PM/QA 리듬을 고정하고, 의사결정 채널(HQ)과 실행 채널(dev/qa/alert)을 분리한다.

## 채널 역할
- HQ: 오너 의사결정/승인 필요 건만 보고
- alert: 진행/지연/리스크/복구 알림
- dev: 구현 진행 및 기술 논의
- qa: 테스트 실행/결과/재검증

## 주간 리듬
- 매주 금요일 19:00 (Asia/Seoul)
- PM + Dev 개발 이슈 리뷰
- 급한 이슈 우선 리스트업
- 개발 가능 범위/일정 확정

## PM 책임
- 오너가 개발하기로 한 항목 기준으로 일정/범위 조율
- 블로커/난이도/지연 리스크 조기 식별
- 실행 가능하도록 작업 단위 정리

## Dev 책임
- 합의된 우선순위대로 구현
- 이슈 발생 시 즉시 PM/alert 공유

## QA 책임
- 구현 완료 건 즉시 검증
- 결함은 dev로 반송, 결과는 qa 채널 기록

## 에스컬레이션 규칙
다음은 HQ로 올린다.
- 범위 변경(축소/연기/분할) 의사결정 필요
- 일정 충돌/리소스 충돌로 우선순위 재결정 필요
- 기술적 제약으로 대안 선택 필요
- Go/No-Go 판단 필요

## 자동화
- Cron 설정: 매주 금요일 19:00 주간 이슈 리뷰 리마인드
- 리마인드 시 PM은 우선순위/일정/리스크를 갱신하고 필요한 의사결정만 HQ에 제출
